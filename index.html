3.2 — Solidity Lanjutan: Events, Inheritance & Keamanan Dasar
⏱ 6–8 Jam+CEI Pattern
▼
⚠️ Pola Keamanan Wajib: Checks-Effects-Interactions (CEI)
Selalu tulis fungsi Solidity dalam urutan: Checks (validasi kondisi) → Effects (update state) → Interactions (panggil contract lain). Urutan ini mencegah reentrancy attack — eksploitasi paling umum yang mencuri jutaan dolar. Contoh salah: update balance setelah transfer. Contoh benar: update balance sebelum transfer. Lihat contoh konkret di solidity-by-example.org/hacks/re-entrancy

🔵 On-Chain Scaling: EIP-4844 & Proto-Danksharding (Live Sejak Maret 2024)
EIP-4844 menambahkan tipe transaksi baru — blob transactions — ke Ethereum mainnet. Blob data tersimpan sementara (~2 minggu) di beacon node, bukan di EVM storage permanen, sehingga biaya L2 untuk posting data turun 5–20×. Ini adalah langkah pertama menuju full danksharding di mana data availability di-shard ke ribuan validator. Relevansi developer: jika kamu deploy ke Arbitrum, Optimism, atau Base, kamu sudah menikmati manfaatnya — tapi memahami mekanismenya penting untuk debugging biaya gas L2 dan mengikuti Ethereum roadmap.
↗ EIP-4844 spec ↗ ethereum.org/en/roadmap/danksharding
Konteks — Evolusi L2 Scaling: Tabel di atas merangkum semua pendekatan scaling yang ada di roadmap.sh sebagai node tersendiri. Di 2025, Rollups (Optimistic + ZK) adalah pendekatan dominan untuk Ethereum — didukung oleh EIP-4844 yang secara dramatis menurunkan biaya data. State Channels dan Plasma relevan untuk dipahami secara historis, bukan untuk diimplementasikan di proyek baru.

Panduan Membaca & Menulis Audit Report
1. Baca Executive SummaryGambaran besar: berapa temuan, severity breakdown, konteks protokol. Ini memberi konteks sebelum masuk detail.
2. Struktur Setiap FindingTitle → Severity → Description → Impact → PoC Code → Recommendation. Hafalkan format ini.
3. Severity MatrixCritical = dana bisa dicuri. High = kerugian signifikan. Medium = fungsi terdampak. Low = best practice / gas.
4. Latihan AktifBaca 1 finding, tutup laporan, tulis ulang dengan kalimat sendiri. Ini membuktikan kamu benar-benar paham.
